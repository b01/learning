# Install Kubernetes Packages

IMPORTANT: These steps MUST be run on each node in the cluster.

1. Complete the [Machine Initialization] and [Install containerd] guides, then
   come back here.
2. Add the Kubernetes GPG key to be trusted on your system:
   ```shell
   curl -fsSL https://pkgs.k8s.io/core:/stable:/v1.34/deb/Release.key | sudo gpg --dearmor -o /etc/apt/keyrings/kubernetes-apt-keyring.gpg
   ```
   NOTE: This is pointing to the version 1.34 repository.
2. Add the appropriate Kubernetes apt repository:
   ```shell
   echo 'deb [signed-by=/etc/apt/keyrings/kubernetes-apt-keyring.gpg] https://pkgs.k8s.io/core:/stable:/v1.34/deb/ /' | sudo tee /etc/apt/sources.list.d/kubernetes.list
   ```
3. Update the apt package index, install kubelet, kubeadm and kubectl, and pin their version:
   ```shell
   sudo apt-get update
   sudo apt-get install -y kubelet kubeadm kubectl
   sudo apt-mark hold kubelet kubeadm kubectl
   ```

### Install ECR Credential Provider

You'll need to build the [ecr-credential-provider] app yourself.

1. Add an IAM managed policy, name it "ecr-credentials-provider" then attach to each k8s node's role:
   ```json
   {
     "Version": "2012-10-17",
     "Statement": [
       {
         "Effect": "Allow",
         "Action": [
           "ec2:DescribeInstances",
           "ec2:DescribeRegions",
           "ecr:GetAuthorizationToken",
           "ecr:BatchCheckLayerAvailability",
           "ecr:GetDownloadUrlForLayer",
           "ecr:GetRepositoryPolicy",
           "ecr:DescribeRepositories",
           "ecr:ListImages",
           "ecr:BatchGetImage"
         ],
         "Resource": "*"
       }
     ]
   }
   ```
2. Install Go:
   ```shell
   GO_VER="1.26.1"
   curl -LO https://go.dev/dl/go${GO_VER}.linux-amd64.tar.gz
   sudo tar -C /usr/local -xzf go${GO_VER}.linux-amd64.tar.gz --strip-components=1
   export GOPATH=/home/ubuntu
   export PATH="${PATH}:/home/ubuntu/bin"
   export CGO_ENABLED=0
   ```
3. Clone and build the `ecr-credential-provider` repository:
   ```shell
   sudo apt install -y make
   git clone https://github.com/kubernetes/cloud-provider-aws.git
   cd cloud-provider-aws
   git checkout -b tag-v1.34.2 v1.34.2
   make ecr-credential-provider
   ecr-credential-provider --version
   ```
4. Move the binary to:
   ```shell
   sudo mv ./ecr-credential-provider /usr/local/bin
   ```
5. Test it works:
   ```shell
   echo '{"apiVersion": "credentialprovider.kubelet.k8s.io/v1","image": "602401143452.dkr.ecr.us-west-2.amazonaws.com/amazon-k8s-cni-init","kind":"CredentialProviderRequest"}' | /usr/local/bin/ecr-credential-provider get
   ```
   NOTE: If it hangs, then you'll likely need to allow IPv4 internet access.
   See [Fix Internet Access for IPv4].
6. Make a configuration from the sample [credential-provider-config.yaml] file
   on control-plane-01.
   This is passed into the kubeadm init config to let the `kubelet` know
   how to use AWS Credential Provider.
   ```shell
   vi credential-provider-config.yaml
   sudo cp credential-provider-config.yaml /var/lib/kubelet
   ```
7. Copy that file to the worker nodes:
   ```shell
   scp credential-provider-config.yaml worker-01:~/
   scp credential-provider-config.yaml worker-02:~/
   ```
   NOTE: Go onto each worker node and move that file into the `/var/lib/kubelet`
   directory.
