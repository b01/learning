# Deploy an AWS Load Balanced Service

This takes a deployment, places a Kubernetes Service in front of it, which is
backed by an AWS load balancer simply by using the type `LoadBalancer` for the
service.

NOTE: This allocates a real AWS Load Balancer, which cost money. As long as we
don't leave it running for more than 10 minutes, we should not be charged more
than a few cents on the dollar.

1. Make a deployment using Nginx:
   ```shell
   cat <<TXT | sudo tee deploy-nginx-lb-app-1.yaml
        apiVersion: apps/v1
        kind: Deployment
        metadata:
          name: nginx-lb-app-1
        spec:
          selector:
            matchLabels:
              run: nginx-lb-app-1
          replicas: 2
          template:
            metadata:
              labels:
                run: nginx-lb-app-1
            spec:
              containers:
              - name: nginx-lb-app-1
                image: nginx
                ports:
                - containerPort: 80
   TXT
   kubectl apply -f deploy-nginx-lb-app-1.yaml
   kubectl get pods -l run=my-nginx -o wide
   ```
2. Create a Service of type `LoadBalancer`:
   ```shell
   kubectl expose deployment -n default nginx-lb-app-1 --port=80 \
     --target-port=80 --name=nginx-lb-app-1-svc --type=LoadBalancer
   ```
3. Get the Service's IP address:
   ```shell
   kubectl describe services nginx-app
   kubectl get pods -l run=nginx-lb-app-1 -o custom-columns=POD_IP:.status.podIPs
   ```
   NOTE: You should be able to ssh into any node in your cluster and use a tool
   such as curl to make queries against both IPs.

## Cleanup

Load Balancers cost money. Please be sure that they are cleaned up afterwards.
1. Go to the `EC2` service.
2. Go to "Load Balancers"
3. In the same region as your EC2 instance locate any load balancers associated
4. with your service and wait a few minutes to see if they are delete, if not,
   then manually delete it.
