# Control Plane

The control-plane is the cluster. It is defined as all the components needed to
run a cluster. Which make global decisions about the cluster.

Control plane components can be run on any machine in the cluster. Where they
run is not too important, however, setup scripts typically start all control
plane components on the same machine.

Taints are applied to machines designated to be control-planes. They are
applied to prevent scheduling non-system Pods on them. But its considered a bad
thing if you run normal workloads on a CP. Its more of a security/stability
precaution.

## Components

Components can be ranked by which should be started from first to last.
`kubelet` it starts all other components, Then `ectd`, because the
`kube-apiserver` depends on it, and all the others depend on the
`kube-apiserver`.
