# Install AWS Load Balancer Controller

The controller runs on the worker nodes and requires access to AWS APIs in order
to deploy ALBs and NLBs. For that IAM permissions are retrieved from the EC2
IMDS to gain AWS credentials.

When using this option, IMDS must be enabled. The controller retrieves the
instance credentials using IMDS.

1. Download the IAM policy to be attached to the instance profile of all nodes:
   ```shell
   curl -L -o iam-policy.json https://raw.githubusercontent.com/kubernetes-sigs/aws-load-balancer-controller/v3.0.0/docs/install/iam_policy.json
   ```
2. Update the policy to include the following condition for the VPC allowed:
   ```json
   {
     "Effect": "Allow",
     "Action": [
       "ec2:AuthorizeSecurityGroupIngress",
       "ec2:RevokeSecurityGroupIngress"
     ],
     "Resource": "*",
     "Condition": {
       "ArnEquals": {
           "ec2:Vpc": "arn:aws:ec2:<REGION>:<ACCOUNT-ID>:vpc/<VPC-ID>"
       }
     }
   }
   ```
3. Attach the policy to each node's (EC2 instance) IAM role.
4. Install with the manifest from the samples
   [aws-cloud-controller-manager.yaml].
   ```shell
   wget https://raw.githubusercontent.com/b01/learning/refs/heads/main/kubernetes/samples/aws-cloud-controller-manager.yaml
   ```
5. Edit the configuration to configure the `aws-cloud-controller-manager`
   DaemonSet for IPv6:
   1. Add a cloud-config.conf
      ```yaml
        containers:
           - name: aws-cloud-controller-manager
             # ...
             volumeMounts:
               - name: cloud-config
                 mountPath: /etc/kubernetes/cloud-config.conf
                 subPath: cloud-config.conf
      # ...
      ---
      apiVersion: v1
      kind: ConfigMap
      metadata:
        name: cloud-config
        namespace: kube-system
      data:
        cloud-config.conf: |
          [Global]
          NodeIPFamilies=ipv4
          NodeIPFamilies=ipv6
      ---
      ```
   2. Add an argument to enable the dual stack feature:
      ```yaml
      containers:
        - name: aws-cloud-controller-manager
          args:
            # ...
            - --feature-gates=CloudDualStackNodeIPs=true
      ```
6. Deploy it
   ```shell
   kubectl apply -f aws-cloud-controller-manager.yaml
   ```

You should see the CoreDNS container trend to "Pending".

NOTE: If you do not specify any annotations in your Kubernetes Service, it will
default to Classic Load Balancer (CLB). You MUST add annotations in order
to use other types Application Load Balancer (ALB), or Network Load Balancer
(NLB). See [Service annotations] for more details.

The LBC will only deploy a dual-stack NLB or a dual-stack ALB upon
consuming corresponding kubernetes service/ingress definition annotated with:
`alb.ingress.kubernetes.io/ip-address-type: dualstack` and
`alb.ingress.kubernetes.io/target-type: ip`
