# CKA Exam Guide v1.32

The [Certified Kubernetes Administrator (CKA) Exam Curriculum] outlines what
will be covered on the exam.

This document expands on that document by linking to resources that help you
comprehend each section, and includes examples that you should be able to try;
provided you have access to a Kubernetes cluster.

This document is organized the same as the CKA exam curriculum, placing
the resources under each section. Going from top-to-bottom, you shouldn't have to worry that you skipped anything.

Note that most, if not all, of the resources linked here are free.

Resources link to notes and exercises within this repository that then cite official documents or other external resources that helped to cover the subjects.

If you notice time limits, know these are meant to serve as a reference on how fast you should perform the task. This helps ensure you can keep pace on the real exam and complete it in enough time.

## 10% - Storage

Review all the content of [Storage] in this guide to get you prepared for the
CKA and interviews.

1. Implement [storage classes] and [dynamic volume provisioning]
2. Configure [volume types], [access modes] and [reclaim policies]
3. Manage [persistent volumes] and [persistent volume claims]
   1. [Configure a Pod to Use a PersistentVolume for Storage]

## 15% - Workloads and Scheduling

Review [Workloads and Scheduling].

1. Understand [application deployments] and how to perform [rolling update]
   and [rollbacks]
   1. [Using kubectl to Create a Deployment]
   2. [Perform a Rolling Update on a DaemonSet]
2. Use [ConfigMaps] and [Secrets] to configure applications
   1. [Configure a Pod to Use a ConfigMap] (exercise)
   2. [Distribute Credentials Securely Using Secrets] (exercise)
3. Configure [workload autoscaling]
4. Understand the primitives used to create robust, [self-healing], application
  deployments.
5. Configure [Pod admission] and [scheduling] (limits, node affinity, etc.)
   1. [Taints and Tolerations]
   2. [Requests and limits]

## 20% - Servicing and Networking

1. Understand [connectivity between Pods]
   1. [Connecting Applications with Services] (exercises)
   2. [Job with Pod-to-Pod Communication] (exercises)
2. Define and enforce [Network Policies]
3. Use ClusterIP, NodePort, LoadBalancer [service types] and [endpoints]
4. Use the [Gateway API] to manage [Ingress] traffic
5. Know how to use [Ingress controllers] and Ingress resources
6. Understand and use [CoreDNS]

## 30% - Troubleshooting

1. Troubleshoot clusters and nodes
   1. [Troubleshooting Clusters]
2. Troubleshoot cluster components
3. Monitor cluster and application resource usage
   1. [Tools for Monitoring Resources]
4. Manage and evaluate container output streams
   1. [Logging Architecture]
   2. [Container Runtime Interface streaming explained]
5. Troubleshoot services and networking

## 25% - Cluster Architecture, Installation and Configuration

1. Manage [role based access control (RBAC)]
2. Prepare underlying infrastructure for installing a Kubernetes cluster
3. Create and manage Kubernetes clusters using kubeadm
4. Manage the lifecycle of Kubernetes clusters
5. Implement and configure a highly-available control plane
6. Use Helm and Kustomize to install cluster components
7. Understand extension interfaces (CNI, CSI, CRI, etc.)
8. Understand CRDs, install and configure operators
