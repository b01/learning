# Guides: Generate an API Token

This guide walks you through the process of generating an API token. This is
an alterative to using a certificate for authentication.

Following these steps will generate an API token for the "current context"
user of the cluster.

1. Create a Service Account in your desired namespace.

   ```bash
   kubectl create serviceaccount my-service-account
   ```

2. Bind the Service Account to a Role that has the necessary permissions

   ```bash
   kubectl create clusterrolebinding my-service-account-binding \
     --clusterrole=cluster-admin \
     --serviceaccount=default:my-service-account
   ```
   NOTE: Binding is how a Service Account/User/Group is granted permissions.
   This uses a cluster-role that comes with most clusters. You'll learn how to
   make your own roles and list all roles available in a cluster.
3. Retrieve the Token for the Service Account:
   1. Get the name of the secret associated with the Service Account.

      ```bash
      kubectl get secrets | grep my-service-account
      ```

   2. Look for a secret with a name that includes `my-service-account`. Then,
      describe the secret to get the token:

      ```bash
      kubectl describe secret my-service-account-token-xxxx
      ```

   3. You'll find the `token` field in the output, which contains the API token.
4. Update the Kubernetes context to use the new service account token:
   1. Get the current context name:

      ```bash
      kubectl config current-context
      ```
   2. Then, set the user credentials in the current context to use the token:

      ```bash
      kubectl config set-credentials my-user --token=YOUR_TOKEN
      ```
   3. Replace `YOUR_TOKEN` with the actual token value.
5. Finally, update the context to use the new user:

   ```bash
   kubectl config set-context $(kubectl config current-context) --user=my-user
   ```
