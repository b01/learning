# Upgrade with kubeadm

How to upgrade a cluster with `kubeadm`.

## Upgrade Worker Node

1. Login to the control-plane and cordon the node. Move/shutdown any daemons or
   state-sets.

   NOTE: If the node has not joined the cluster yet, then skip this
   step as you may only need to upgrade the packages.
2. Stop the kubelet.
3. Upgrade `kubeadm`, `kubectl`, and `kubelet`.
   1. Remove the hold, if any on them.
   2. Update the repository to the exact version you are upgrading to.
   3. Run:
      ```shell
      sudo apt update
      sudo apt upgrade -y
      ```
4. Install any software required for the CNI or other components, for example,
   in AWS you may need to install `ecr-credential-provider` for example.
5. Start the kubelet.
6. Go back the control-plane terminal and un-cordon the node. If you need to
7. join the node to the cluster, then run:

   ```shell
   kubeadm token create --print-join-command
   ```
8. You can move any daemon/state sets back on the nodes once it's ready.

## Upgrade Control-plan

IMPORTANT: If you only have 1 control-plane, then this will take your cluster
down until the upgrade is completed and you bring the control-plane back online.
