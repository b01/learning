# Build A kubeadm Init Config

1. Log into the control plane with:
   ```shell
   aws ec2-instance-connect ssh --instance-id <instance-id> --os-user ubuntu --private-key-file <private-key-file
   ```
2. Choose which IP CIDR ranges to use for IPv4 and IPv6. You can use the ones
   provided here, which are common in examples you may have seen before.
   The private class for IPv4 and global unicast for IPv6:
   ```text
   IPv4 POD network:     10.244.0.0/16
   IPv4 Service Network: 10.96.0.0/16
   IPv6 POD network:     fd12:3456:789a:1000::/56
   IPv6 Service network: fd12:3456:789a:2000::/112
   ```
3. You'll need to generate a `kubeadm` init config, let's make it quick and use
   the sample template [kubeadm-dual-stack-init-config-aws.yaml.tmpl].

   NOTE: You can also generate an init config, set to defaults, using `kubeadm`
   ```shell
   kubeadm config print init-defaults --component-configs KubeletConfiguration | tee kubeadm-dual-stack-init-config-aws.yaml.tmpl
   ```
4. Review the config `kubeadm`, it should contain `ecr-credential-provider`
   arguments for the `kubelet`. It must pass the
   `--image-credential-provider-config`  and `image-credential-provider-bin-dir`
   options. Along with the feature gate option
   to enable `KubeletServiceAccountTokenForCredentialProviders`. For example:
   ```yaml
   apiVersion: kubeadm.k8s.io/v1beta4
   kind: InitConfiguration
   nodeRegistration:
     kubeletExtraArgs:
       # ...
       - name: image-credential-provider-config
         value: /var/lib/kubelet/credential-provider-config.yaml
       - name: image-credential-provider-bin-dir
         value: /usr/local/bin/ecr-credential-provider
   ---
   apiVersion: kubelet.config.k8s.io/v1beta1
   kind: KubeletConfiguration
   featureGates:
     KubeletServiceAccountTokenForCredentialProviders: true
   ```
5. Continuing on, make sure it includes:
   * the IPV6_ADDR placeholders (you'll fill those in later)
   * the advertisement address set to the IPV6_ADDR placeholder
   * set the cluster endpoint
   * the IPv6 loopback and IPV6_ADDR addresses in the certSANs
   * Pod and Service Network IPv6 CIDRs
   ```yaml
   ---
   apiVersion: kubeadm.k8s.io/v1beta4
   kind: InitConfiguration
   localAPIEndpoint:
     advertiseAddress: ${IPV6_ADDR}
     bindPort: 6443
   nodeRegistration:
     kubeletExtraArgs:
       - name: node-ip
         value: ${IPV6_ADDR},${IPV4_ADDR}
       - # ...
   ---
   apiVersion: kubeadm.k8s.io/v1beta4
   kind: ClusterConfiguration
   kubernetesVersion: 1.34.0
   networking:
     dnsDomain: cluster.local
     podSubnet: fd12:3456:789a:1000::/56,10.244.0.0/16
     serviceSubnet: fd12:3456:789a:2000::/112,10.96.0.0/16
   # For now, you can add a hosts file entry so that control-plane resolves correctly.
   controlPlaneEndpoint: control-plane
   apiServer:
     certSANs:
      - ::1
      - 0:0:0:0:0:0:0:1
      - 127.0.0.1
      - ${IPV6_ADDR}
      - ${IPV4_ADDR}
      - ${HOSTNAME}
      - control-plane
   ```
6. Configure AWS Cloud Controller Manager.

   This shows how to stop the cluster from using the built-in cloud provider
   code in the `kube-controller-manager`, to using the external
   `cloud controller manager`.

   Setting `cloud-provider=external` will make the cluster wait until both the
   `aws-vpc-cni` and the `aws-cloud-controller-manager` are running. Only then
   will `core-dns` begin to run and `kube-proxy` begin to make routes.

   This was proven to work in `IPv4` mode.

   This will fail for `DualStack` mode because the `aws-vpc-cni` DOES NOT
   SUPPORT DUAL STACK configurations and at the time.

   1. For the `aws-cloud-controller-manager` to be able to communicate to AWS
      APIs, you will need to add a few IAM policies to your EC2 instances. For
      control-planes attach [aws-cloud-controller-manager-control-plane.json]
      and for workers attach [aws-cloud-controller-manager-worker.json].
   2. Further review the `kubeadm` init configuration indicates using an
      external cloud controller:
      ```yaml
      ---
      apiVersion: kubeadm.k8s.io/v1beta4
      kind: InitConfiguration
      # ...
      nodeRegistration:
        kubeletExtraArgs:
          # ...
          - name: cloud-provider
            value: external
      ---
      apiVersion: kubeadm.k8s.io/v1beta4
      kind: ClusterConfiguration
      # ...
      controllerManager:
        extraArgs:
          - name: cloud-provider
            value: external
          - name: leader-elect
            value: "true"
          - name: external-cloud-volume-plugin
            value: aws
      ```
7. Let's fill in the IP values using `envsubst`.
   ```shell
   getIps () {
      iface=${1}
      export IPV4_ADDR="$(ip -4 route get 8.8.8.8 | awk '{print $7}')"
      export IPV6_ADDR="$(ip -6 route get 2607:f8b0:4009:816::200e | awk '{print $11}')"
      echo "IPV4_ADDR=${IPV4_ADDR}"
      echo "IPV6_ADDR=${IPV6_ADDR}"
   }
   getIps "ens5"
   envsubst < kubeadm-dual-stack-init-config-aws.yaml.tmpl | sudo tee kubeadm-dual-stack-init-config-aws.yaml
   ```
